## FEATURE 1: PDF Report Generator (/#/reports)

Page that generates professional PDF reports using jspdf + jspdf-autotable (both installed).

### UI Layout:
- Header: "Generate Reports" with FileText icon
- Report type selector (cards):
  1. Monthly Attendance Report
  2. Quarterly Performance Report
  3. School Readiness Report
  4. Visit Coverage Report
- Filters: State dropdown, LGA dropdown (optional), Date range picker (month/quarter selector)
- "Generate PDF" button
- Preview section showing report summary before download

### PDF Report Content (Monthly Attendance):
Page 1 - Cover:
- "EduTrack Nigeria" header
- "Monthly School Attendance Report" 
- State name, Month/Year
- "Prepared by Harmony Digital Consults Ltd"
- Report date

Page 2 - Executive Summary:
- Total schools monitored
- Average student attendance rate (%)
- Average teacher attendance rate (%)
- Schools with 90%+ attendance (count)
- Schools below 70% attendance (count and names)
- Comparison vs previous month

Page 3 - Attendance by LGA (table):
- LGA name, Schools, Avg Student %, Avg Teacher %, Days Reported, Trend (up/down arrow)

Page 4 - Top & Bottom Performers:
- Top 10 schools by attendance rate
- Bottom 10 schools by attendance rate  
- Each row: School name, LGA, Student %, Teacher %, Trend

Page 5 - Visit Coverage:
- Total visits this month
- Schools visited vs not visited
- SSO performance table

Footer on every page: "Generated by EduTrack Nigeria | Harmony Digital Consults Ltd | [date]"

Use jspdf with:
- Colors: #01696F (teal header), #1A1A2E (body text)
- Font size: 20pt title, 12pt section headers, 10pt body
- jspdf-autotable for all tables
- Page numbers

In DEMO MODE: Generate report using the static data from data.ts.
In PRODUCTION MODE: Fetch from Supabase views.

### Sidebar: Add under ANALYTICS group, icon: FileBarChart, title: "Reports", minRole: "subeb_admin"

## FEATURE 2: Resource Request System (/#/resource-requests)

### Two views on the same page (tabs):

Tab 1: "Submit Request" (for head_teacher and above)
- School (auto-filled for head teachers, dropdown for admins)
- Category dropdown: Furniture, Textbooks, Lab Equipment, Building Repair, Toilet/Water, Teaching Aids, ICT Equipment, Other
- Description textarea
- Quantity (number)
- Priority: Low, Medium, High, Urgent
- Submit button

Tab 2: "All Requests" (for subeb_admin and above)
- Table: Date, School, Category, Description, Qty, Priority (colored badge), Status (Pending/Approved/In Progress/Completed/Rejected)
- Filter by: Status, Priority, Category, School
- Admin can change status via dropdown in the table
- Summary cards at top: Total Requests, Pending, Approved, Completed

### Data storage:
In DEMO MODE: Use a React state array with sample data (10-15 demo requests)
In PRODUCTION MODE: Store in Supabase. Create a note/instruction for the user to add a resource_requests table.

### Sidebar: Add under DATA ENTRY group, icon: PackagePlus, title: "Resource Requests", minRole: "head_teacher"

## FEATURE 4: Alert Configuration (/#/alerts)

### Layout:
- Header: "Alerts & Notifications"
- Tabs: "Active Alerts" | "Configure Rules" | "Alert History"

Tab 1: Active Alerts
- List of current active alerts with severity icon (red/yellow/green)
- Demo data: 5-8 sample alerts like:
  - "⚠ 3 schools haven't reported attendance in 5+ days" (Warning)
  - "🔴 Teacher attendance below 50% at Gov't Primary School Badawa" (Critical)
  - "✅ Kano LGA achieved 90% visit coverage this month" (Success)
  - "⚠ Resource request pending for 14 days: Textbooks at LEA School Kawo" (Warning)
- Each alert: severity badge, message, timestamp, "Dismiss" button

Tab 2: Configure Rules
- List of configurable alert rules (UI only — show the concept):
  1. "No attendance report" — trigger after X days (input: 3)
  2. "Low student attendance" — below X% (input: 65)
  3. "Low teacher attendance" — below X% (input: 70)
  4. "Visit coverage" — alert if below X% of schools visited (input: 50)
  5. "Resource request aging" — alert if pending more than X days (input: 14)
- Each rule: toggle on/off, threshold input, notification method checkboxes (Dashboard, SMS, WhatsApp, Email)
- "Save Configuration" button (shows toast in demo mode)

Tab 3: Alert History
- Table of past alerts: Date, Type, Message, Severity, Status (Active/Resolved/Dismissed)
- Demo data: 15-20 historical alerts

### Note at bottom: "SMS and WhatsApp notifications require Africa's Talking integration. Contact Harmony Digital Consults for setup."

### Sidebar: Add under MANAGEMENT group, icon: Bell, title: "Alerts", minRole: "subeb_admin"

## FEATURE 5: PWA Offline Support

### Add to client/index.html:
- PWA manifest (manifest.json)
- Service worker registration

### Create public/manifest.json:
```json
{
  "name": "EduTrack Nigeria",
  "short_name": "EduTrack",
  "description": "School Operational Excellence Dashboard",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#01696F",
  "theme_color": "#01696F",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

### Add offline indicator component:
- When navigator.onLine is false, show a yellow banner: "You're offline — data will sync when connection returns"
- Style the attendance and visit forms to show "Saved locally (pending sync)" when submitted offline

### Note: Full service worker implementation would require a build tool config change. For now, add the manifest and the offline detection UI. The actual caching can be added incrementally.

## TECHNICAL RULES:
- All new pages use shadcn/ui components
- All tables use the Table component
- Charts use Recharts
- Mobile responsive
- Dark mode support
- Role-aware (use useAuth hook)
- Demo mode support (all features must work without Supabase)
- Use useHashLocation for any navigation
- Add data-testid attributes to interactive elements
